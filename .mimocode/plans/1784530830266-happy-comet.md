# Plan: Hero Image Animations (No Rings)

## Goal
Replace the cached ring animations with new, different animations on the hero image side. The user likes the red offset border and corner brackets — keep those. Add subtle motion to make the image area feel alive.

## Current State
- Hero has a 2-column layout: text left, image right
- Image has: black border, red offset border (`vb-hero__image-border`), 4 black corner brackets
- Ring CSS was removed from `frontend.scss` but may be cached in browser
- User wants NEW animations (not rings) on the image side

## Changes

### 1. CSS — `resources/css/frontend.scss`

Add these animations to the image area:

**a) Dashed border animation** — An SVG-style dashed border that rotates slowly around the image
- Use a `::before` pseudo-element on `.vb-hero__image-wrap`
- Dashed border with `stroke-dasharray` animation (CSS-only approach using `border-style: dashed` + rotation)
- Subtle, slow rotation (20-30s per revolution)

**b) Floating animation** — Gentle vertical oscillation on the image container
- `.vb-hero__image-wrap` gets a `translateY` keyframe (0 → -8px → 0)
- Duration: 4-5s, ease-in-out, infinite

**c) Corner bracket pulse** — Subtle scale pulse on the corner brackets
- Each corner bracket gets a gentle scale pulse (1 → 1.05 → 1)
- Staggered timing so they don't all pulse at once

**d) Gradient shimmer** — A diagonal gradient sweep across the image (like a light reflection)
- Use `::after` pseudo-element on `.vb-hero__image`
- Diagonal white gradient that moves across on hover or on a loop
